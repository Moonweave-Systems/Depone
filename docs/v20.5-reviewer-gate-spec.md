# V20.5 Reviewer Gate Spec

Status: implemented first reviewer gate in `scripts/dwm_review_gate.py`.

## Purpose

V20.5 adds an independent release-review gate after V20 release-candidate
hardening. It reviews generated `release-status.json` evidence and writes a
separate review artifact before any later publication or live adapter smoke.

## Non-Goals

- do not publish a package,
- do not execute live Codex or Claude adapters,
- do not treat the release generator as its own reviewer,
- do not mutate prior release evidence.

## Contract

The reviewer accepts only release status artifacts with:

- `status: accepted`,
- `decision: release-candidate`,
- all required gates present,
- a recomputed `gate_hash` matching `gates`,
- no missing compatibility, security, migration, or release corpus gate.

## Outputs

`out/release-review/<id>/` contains:

- `.dwm_review_gate-owned.json`,
- `review.json`,
- `review.md`,
- `status.json`.

## Evaluation Fixtures

- positive: accepted release is reviewed and approved,
- negative: stale release hash is blocked,
- negative: blocked release source is rejected,
- negative: missing required gate is rejected.

Failure codes:

- `ERR_REVIEW_GATE_STALE_RELEASE`,
- `ERR_REVIEW_GATE_RELEASE_NOT_ACCEPTED`,
- `ERR_REVIEW_GATE_MISSING_GATE`.
