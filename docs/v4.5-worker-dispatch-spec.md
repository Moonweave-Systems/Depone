# V4.5 Worker Dispatch Spec

Status: first slice implemented
Date: 2026-06-15

## Purpose

V4.5 bridges V4 scheduling to future worker execution without launching any
worker. It consumes one trusted V4 packet and emits a deterministic dispatch
bundle that can be handed to a human, Codex, OMX, or a later execution adapter.

The dispatch bundle is evidence, not execution. It proves which packet, prompt,
worker IDs, expected outputs, source V4 status, and hashes were prepared.

## Command Contract

```bash
python scripts/dispatch_worker.py --packet out/v4/<run_id>/packets/<packet>.packet.json --out out/v4.5/<run_id>
python scripts/dispatch_worker.py --resume out/v4.5/<run_id>
python scripts/dispatch_worker.py --self-test
```

## Accepted Inputs

V4.5 accepts only a packet under an owned V4 output directory with:

- `.orchestrate_workflow-owned.json`,
- `status.json` with `status: scheduled`,
- `schedule.json` selecting the packet phase,
- packet and prompt hashes matching V4 status snapshots.

## Output Model

Accepted dispatch writes:

```text
out/v4.5/<run_id>/
├── .dispatch_worker-owned.json
├── dispatch.json
├── packet.json
├── prompt.md
├── hashes.json
├── status.json
└── resume.md
```

`dispatch.json` records `status: prepared`, `mode: emit-only`, source V4 paths,
packet ID, phase ID, worker IDs, expected outputs, and stop conditions.

## Non-Goals

- Do not execute the packet.
- Do not spawn subagents.
- Do not create or merge worktrees.
- Do not call Codex CLI, OMX, network APIs, paid APIs, or external services.
- Do not commit, push, install dependencies, deploy, access secrets, delete
  files, or rewrite history.

## Resume Behavior

`--resume` recomputes the trusted V4 source packet, prompt, dispatch, and hashes.
It writes only `status.json` and `resume.md`.

Resume returns:

- `prepared` with `resume_state: resumable` when all artifacts still match,
- `invalid` with `ERR_DISPATCH_ARTIFACT_MALFORMED` when dispatch, packet copy,
  prompt copy, or hashes were tampered,
- `blocked` when the V4 source is no longer trusted.

## Release Criteria

The slice is `keep` only if:

- `python scripts/dispatch_worker.py --self-test` passes,
- a real V4 dogfood packet prepares a dispatch bundle,
- clean dispatch resume is resumable,
- tampered dispatch artifacts invalidate resume,
- no execution or side effects are introduced.
