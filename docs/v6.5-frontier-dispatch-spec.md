# V6.5 Frontier Dispatch Spec

Status: first slice implemented
Date: 2026-06-15

## Purpose

V6.5 is the first loop-back slice. It consumes a trusted V6 frontier and emits
an owned dispatch bundle for the selected next phase.

The loop is:

```text
V6 frontier packet
-> V6.5 dispatch bundle
-> future worker result adapter
-> V7.5 frontier-result review
-> future runtime ingestion
```

V6.5 does not execute the selected packet. It only makes the next phase
dispatchable.

## Workflow Design

Source plan: `docs/v6.5-frontier-dispatch.workflow.plan.json`.

Patterns:

- Sequential
- Resume And Cache
- Human Gate

Phases:

1. Frontier validation: resume V6 and validate selected packet/prompt hashes.
2. Dispatch preparation: write `dispatch.json`, `packet.json`, `prompt.md`,
   `hashes.json`, `status.json`, and `resume.md`.
3. Resume verification: recompute source and dispatch artifacts.

## Command Contract

```bash
python scripts/dispatch_frontier.py --frontier out/v6/<run_id> --out out/v6.5/<run_id>
python scripts/dispatch_frontier.py --resume out/v6.5/<run_id>
python scripts/dispatch_frontier.py --self-test
```

## Accepted Inputs

V6.5 accepts only:

- an owned V6 directory,
- `status.json` with `status: frontier-ready`,
- clean V6 resume,
- exactly one selected frontier packet for the first slice,
- selected packet and prompt hashes matching V6 state/status snapshots.

V6.5 rejects:

- stale or malformed V6 artifacts,
- `workflow-complete`, `invalid`, or non-resumable V6 states,
- zero or multiple selected frontier packets in the first slice,
- selected packet/prompt hash mismatch,
- symlinked or outside-`out/v6.5` output paths.

## First Slice Rules

For the dogfood result, V6.5 should prepare a dispatch for
`release_decision`.

Prepared status requires:

1. V6 status is `frontier-ready`.
2. V6 resume is `resumable`.
3. V6 selected phase is exactly `release_decision`.
4. Selected packet and prompt match V6 snapshots.
5. Dispatch output is owned and hash-bound.
6. Resume recomputes the same source, dispatch, packet, prompt, and hashes.

## Non-Goals

- Do not execute the selected packet.
- Do not call Codex CLI, OMX, subagents, network APIs, or paid APIs.
- Do not produce worker outputs.
- Do not review or ingest the selected phase result.

## Release Criteria

The slice is `keep` only if:

- `python scripts/dispatch_frontier.py --self-test` passes,
- dogfood dispatch over `out/v6/v32-semantic-dogfood` returns `prepared`,
- clean resume returns `resume_state: resumable`,
- tampered packet copy invalidates resume,
- no worker execution or backend execution is introduced.
